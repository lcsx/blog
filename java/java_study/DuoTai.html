# 1 多态

多态（英语：polymorphism）

多态是同一个行为具有多个不同表现形式或形态的能力。

多态就是同一个接口，使用不同的实例而执行不同操作

多态的必要条件

  1. 继承;
  2. 重写;
  3. 父类引用指向子类对象.

多态的优点

  1. 消除类型之间的耦合关系
  2. 可替换性
  3. 可扩充性
  4. 接口性
  5. 灵活性
  6. 简化性

# 1.1.2 接口

问题引出 Java中只支持单继承，

  1. 那如何实现一个类型中兼容多种类型特征呢？
  2. 多种不同类型，如何有相同的特征呢？

可以通过接口实现

接口（interface）是一个抽象类型，是抽象方法的集合，不是类。

接口通常已interface来声明。一个类通过继承接口的方式来继承接口的抽象方法。除非实现接口的类是抽象类，否则该类要定义接口中的所有方法。

接口无法被实化，因为接口中的方法都是抽象的，是没有方法体的

# 接口没有构造方法。接口不饿能包含成员变量，除了static和final变量

接口的特性

  1. java中没有方法体的方法需要abstract声明，但接口是隐式抽象的，当声明一个接口的时候，不必使用abstract关键字。
  2. 接口中的方法都是公有的public。
  3. 接口不关心类的内部数据，也不关心类里的实现方法的细节，ta之规定这些类里必须提供某些方法
  4. 接口里的方法都需要被实现。若是当前实现类，不完全实现接口的方法，可以将当前实现类设为抽象类abstract，然后由该类的子类来实现未实现的方法。谁类继承，谁来实现。
  5. 接口抽象方法不能带有主体，没有{}
  6. 接口不能实例化，new XX();

接口的通俗理解“空调”——“遥控器” 可以把“节能”作为一种标准，或者说节能就是一个“接口”，这个接口中有一个方法，叫做变频方法，任何空调，如果要称得上叫做节能空调的话，那么必须实现“节能”这个接口，实现“节能”这个接口，也就必须实现“节能”接口中规定实现的“变频”方法， 这样才算是真正的实现了“节能”这个接口，实现了“节能”这个功能。

# 当某个空调实现了“节能”接口后，这个空调就具备了节能的功能，那么我们也可以不用空调类的引用指向空调对象， 我们可以直接使用一个“节能”接口类型引用的“遥控器”去指向“空调”，虽然这个“遥控器”上面只有一个按键，只有一个“变频”的方法， 但是“遥控器”所指向的空调是实现了“节能”这个接口的，是有“变频”方法的实现的，我们用这个只有一个“变频”方法的遥控器去命令空调调用“变频”方法，也是行得通的

# 2.2 接口的实现

当类实现接口的时候，类要实现接口中所有的方法。否则，类必须声明为抽象的类。

类使用implements关键字实现接口。在类声明中，Implements关键字放在class声明后面。

一个接口能继承另一个接口，和类之间的继承方式比较相似。接口的继承使用extends关键字，子接口继承父接口的方法。

package com.lcs.tel;

public abstract interface Usb {
    //接口中抽象方法可以不写abstract
    public void read();
    //写全是
    //public abstract void read();

}

// ...implements 接口名称[, 其他接口名称, 其他接口名称..., ...] ...
//一个接口能继承另一个接口，和类之间的继承方式比较相似。接口的继承使用extends关键字，子接口继承父接口的方法。
// 接口的多继承
// public interface Hockey extends Sports, Event{}
//============================
import com.lcs.tel.Iphoto;
import com.lcs.tel.Usb;
//A类实现Usb接口
public class A implements Usb, Iphoto {
    @Override
    public void read(){
        System.out.println("usb a");
    }

    @Override
    public void takePhoto(){
        System.out.println("paizhao");
    }
}

接口

abstract，使为抽象类，get2()方法为实现

package com.lcs.tel;
// 如果在当前类中不想实现Inet全部的方法，即只实现部分Inet的方法，需要将当前类调整为 抽象类，+ abstract
public abstract class B implements Inet{
    private String name;
    private int age;

    @Override
    public void get1() { }

    public B(String name,int age){
        this.name = name;
        this.age = age;
    }
    public void showInfo(){
        System.out.println("名字："+this.name+"\n年龄："+this.age);
    }
}

子类实现接口剩下的方法

package com.lcs.tel;

    @Override
    public void get2() { }

# 2.5 接口中的defalut 与 static

# 2.5.1 defalut

接口里的方法是必须要要实现的，你不想实现接口方法，但你由不得不实现这个方法。

java1.8中提供一个 默认实现 方式，不想改写接口的方法时。

package com.lcs.tel;

public interface Inet {
    public void get1();

    //default 是由方法体的
    default void connection(){
        System.out.println("接口的默认连接");
    }

    static void stop(){
        System.out.println("接口的静态方法");
    }
}

package com.lcs.tel;

public class C implements Inet{
    @Override
    public void get1() { }

    @Override
    public void connection(){
        Inet.super.connection(); //调用接口的默认方法
    }
}

import com.lcs.tel.C;
import com.lcs.tel.Inet;

public class test {
    public static void main(String[] args) {
        C c1 = new C();
        c1.connection();
        Inet.stop();
    }
}

# 2.5.3 接口变量与父类变量重名

接口方法与实现类的父类方法同名，实现类中没有该方法，则调用的是父类的方法。但是，对于接口变量与父类变量同名时，需要在实现类声明同名变量

interface One{
    final int TEMP = 10;
    default void connection(){
        System.out.println("接口的默认连接");
    }
}

interface Two{
    static int TEMP = 20;
    void connection();
}

class B{
    public static final int TEMP = 30;
    public void connection(){
        System.out.println("B中的connection");
    }
}
public class test extends B implements One,Two{
    public int TEMP = 40;
    public static int LCS = 50;
    public void show1(){
        System.out.println(TEMP);
    }
    public static void main(String[] args) {

        test t1 = new test();
        t1.connection();  //B中的connection
        t1.run();

        One o1 = new test();
        o1.connection();  //B中的connection

        /*
        * o1.run();  // One指向子类实例，但只有接口定义的方法，调用的是实现类或实现类的父类的方法。没有子类的其他方法。
        * */

        //One o2 = new B();  //报错  不兼容的类型: B无法转换为One

        Two two1 = new test();
        two1.connection();  //B中的connection

        /*
        * 接口方法与实现类的父类方法同名，实现类中没有该方法，则调用的是父类的方法。但是，对于接口变量与父类变量同名时，需要在实现类声明同名变量
        * System.out.println(t1.TEMP);  //对TEMP的引用不明确,B 中的变量 TEMP 和 One 中的变量 TEMP 都匹配
        * 解决 实现类中声明同名变量
        */


        //System.out.println(TEMP); //无法从静态上下文中引用非静态 变量 TEMP
        //转成静态变量即可引用
        System.out.println(LCS);  //50
    }

    public void run(){}
}

# 2.6 接口的继承

java类是单继承的，但接口可以多继承。但实现类要实现接口的所有方法。

当前类不想完全实现接口的方法，需要转为抽象类。

当子接口的多个父接口有重名抽象方法，需要在子接口声明那个接口

//写在一个文件里

interface IFather{
    default void connection(){
        System.out.println("IFather中的connection");
    }
}
//public interface IFather{
//    default void connection(){
//        System.out.println("IFather中的connection");
//    }
//}
// 写在一个文件里， interface前不加 public
// 因为，类IFather是公共的, 应在名为 IFather.java 的文件中声明

interface IFather2{
    default void connection(){
        System.out.println("IFather2中的connection");
    }
}

interface ISon extends IFather2{
    void run();
    default void connection(){
        System.out.println("ISon中的connection");
    }
}

public class test implements ISon{
    public void run(){}
    public static void main(String[] args) {
        test t1 = new test();
        t1.connection(); //实现类中未重写connection ，输出的是 ISon中的connection
    }
}
