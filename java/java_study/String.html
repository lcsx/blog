# 2.1 字符串与byte数组间的转换

网络传输数据是通过二进制字节进行，接受数据需要将byte转为字符串。

import imooc.lcs.test.HException;
import sun.nio.cs.ext.GBK;

import java.io.UnsupportedEncodingException;
import java.nio.charset.StandardCharsets;

public class Test1 {
    public static void main(String[] args) throws UnsupportedEncodingException {
       String st1 = "JAVA 编程 基础";
//       byte[] arrs = st1.getBytes("UTF-8");
       byte[] arrs = st1.getBytes("GBK");
       for (int i=0;i< arrs.length;i++){
           System.out.print(arrs[i]+" ");   //UTF-8:74 65 86 65 32 -25 -68 -106 -25 -88 -117 32 -27 -97 -70 -25 -95 -128
                                            //GBK:74 65 86 65 32 -79 -32 -77 -52 32 -69 -7 -76 -95
       }
        System.out.println("============");
        String str2 = new String(arrs,"GBK");  //保持字符集一致
        System.out.println(str2);
    }
}

# 4 String的不可变性

所谓可变：就是创建了新的对象，所指向的内存空间不变。

String类是final类型的。final修饰引用类型，它保存的仅仅是一个引用，final只保证这个引用类型变量所引用的地址不会改变，也就是一直引用同一个对象，但这个对象完全可以发生改变。

当我们使字面量进行创建字符串时，会在常量池中查找是否有相同值的字符串，如果有则会返回其引用，否则会创建新的字符串，并返回新字符串的引用。

为什么String要设计成不可变

主要是从缓存、安全性、线程安全和性能等角度出发的。

缓存 字符串是使用最广泛的数据结构。大量的字符串的创建是非常耗费资源的，所以，Java提供了对字符串的缓存功能，可以大大地节省堆空间。

安全性： 字符串在Java应用程序中广泛用于存储敏感信息，如用户名、密码、连接url、网络连接等。JVM类加载器在加载类的时也广泛地使用它。 当我们在程序中传递一个字符串的时候，如果这个字符串的内容是不可变的，那么我们就可以相信这个字符串中的内容。

不可变性的缺点 需要定义其他变量来存储计算结果，如果是复杂的字符串处理，会生成很多中间变量。可以通过StringBuilder处理。

# 5 字符串StringBuilder

String和StringBuilder的区别：

  1. String具有不可变性；
  2. StringBukder不具备可变性。

当频繁操作字符串，用String会产生大量的中间变量和常量池大量的废弃数据,推荐使用StringBuilder可以避免。

StringBuilder和StringBuffer 两者基本相似，StringBuffer是线程安全，而StringBuilder没有。所以StringBuiler性能略高。
