# 内部类

# 1.1 什么是内部类

在java中，可以将一个类定义在另一个类里面或者一个方法里面，这样的类称内部类。

# 为什么要内部类？ 内部类隐藏在外部类之内，更好的实现了信息隐藏。

# 1.2 内部类的分类

  1. 成员内部类
  2. 静态内部类
  3. 方法内部类
  4. 匿名内部类

# 1.2.1 成员内部类

成员内部类，也称普通内部类

  1. 内部类在外部使用时，无法直接实例化，需要借助外部类信息才能完成实例化；
  2. 内部类的访问修饰符，可以任意，但是访问范围会收到影响；
  3. 内部类可以直接访问外部类的成员；如果出现同名属性，优先访问内部类中定义的；
  4. 可以使用 外部类 .this.成员 的方式，访问外部类中同名的信息；
  5. 外部类访问内部类信息，需要通过内部类实例，无法直接访问；
  6. 内部类编译后.class文件命名：外部类$内部类.class
  7. 内部类中是否可以包含与外部类相同方法签名
package com.lcs.tel;

//外部类
public class Person {
    public int age;
    public Heart geatHeart(){
        return new Heart();
    }

    //成员内部类
    public class Heart{
        public void beat(){
            System.out.println("心跳");
        }
    }
}

跨包调用时，test类需要public new 外部类.new 内部类 外部实例.new 内部类

import com.lcs.tel.Person;

public class test {
    public static void main(String[] args) {
        Person p1 = new Person();
        p1.age = 18;

        //获取内部类实例对象，方式1：new 外部类.new 内部类
        Person.Heart mh = new Person().new Heart();

        //方式2: 外部类对象.new 内部类
        Person.Heart mh1 = p1.new Heart();

        //方式3：自定义个方法 外部类、外部对象   .自定义获取方法
        Person.Heart mh2 = new Person().geatHeart();
        Person.Heart mh3 = p1.geatHeart();
    }
}

# 1.2.2 静态内部类

static修饰的内部类

总结：1、静态内部类中可以写哪些内容

1）匿名代码块

2）静态代码块

3）静态变量和非静态变量

4）静态方法和非静态方法

注意：不能在静态内部类中写抽象方法

2、外部类如何调用静态内部类中的属性和方法

1）外部类可以通过创建静态内部类实例的方法来调用静态内部类的非静态属性和方法

2）外部类可以直接通过“ 外部类.内部类.属性（方法）” 的方式直接调用静态内部类中的静态属性和方法

3、静态内部类如何调用外部类的属性和方法

1）静态内部类可以直接调用外部类的静态属性和方法

2）静态内部类可以通过创建外部类实例的方法调用外部类的非静态属性和方法

4、如何创建静态内部类实例

1）在非外部类中：外部类名.内部类名 name = new 外部类名.内部类名();

2）在外部类中：内部类名 name = new 内部类名();

package com.lcs.tel;


public class Outer {
    //定义一个实例变量和一个静态变量
    private int a;
    private static int b;
    //定义一个静态方法和一个非静态方法
    public static void say(){}
    public void test(){
        //在外部类中调用内部类的属性和方法
        Outer.Inner.c = 1;            //可以通过静态内部类的全类名来调用静态内部类的静态属性（外部类名.静态内部类名.属性）
        Outer.Inner.go();            //可以通过静态内部类的全类名来调用静态内部类的静态方法（外部类名.静态内部类名.方法）
        //Outer.Inner.walk();        //不能通过类静态内部类的全类名来调用内部类的非静态属性和方法
        Inner inner = new Inner();
        inner.d = 1;
        inner.walk();                    //可以通过创建内部类实例来调用静态内部类的非静态属性和方法
    }
    //静态内部类
    public static class Inner{
        //在静态内部类中定义一个静态变量和一个实例变量
        static int c;
        int d;
        //定义一个匿名代码块和一个静态代码块
        {}
        static{}
        //定义一个静态方法和一个普通方法
        public static void go(){}
        public void walk(){
            //在静态内部类中调用外部类的属性和方法
            int f = b;                     //可以直接调用外部类的静态属性
            say();                        //可以直接调用外部类的静态方法
            //int e = a;                 直接调用外部类的非静态属性出错编译出错
            //test();                    直接调用外部类的非静态方法时编译出错
            Outer outer = new Outer();
            int e = outer.a;            //可以通过创建外部类实例来调用外部类的非静态属性
            outer.test();                //可以通过创建外部类实例来调用外部类的非静态方法
        }
    }
}

类的内部不能直接写代码，代码只能写在方法内部

package com.lcs.tel;

//外部类
public class Person {
    public int age = 22;
    public void eat(){
        System.out.println("吃吃吃");
    }
    //静态内部类
    static class Heart{
        public int age = 18;

        // 1.想调用非静态成员，外部类非static的方法，需要通过对象实例访问
        Person p1 = new Person();

        //p1.eat(); 报错 java: 需要<标识符>  :类的内部不能直接写代码，代码只能写在方法内部，作为方法体。main是很特殊的⽅法。
        public void lcs(){
            p1.eat();  //就可以调用了
        }



    }
}

# 1.2.3 方法内部类

也成局部内部类，定义在外部类方法中的内部类

  1. 方法内定义的局部变量智能在方法里使用
  2. 方法内不能定义静态成员
  3. 和方法内部成员使用规则一样，不可以用 public/private/protected/static 修饰，可以抽象类 abstract
  4. 类中可以包含final、abstract修饰的成员
//外部类
public class Person {
    public int age = 22;
    public void eat(){
        System.out.println("吃吃吃");
    }

    public Object getHeart(){
        //定义在方法内部的类
        class Heart{
            public final int age = 6;
            int temp = 22;

            public final void say(){
                System.out.println("hello");
            }

            public String beat(){
                //调用外部类成员
                new Person().eat();
                return "要加油";
            }
        }
        //return new Heart(); 一般很少直接返回实例，一般返回实例的方法
        // 因为：
        // 外部实例执行 p1= Person.getHeart()时，返回new Heart()
        // 但 p1不能调用方法内部类的其他方法了 如say、beat方法
        // 所以要想调用方法内部类中的方法，需要return具体的方法,如下
        return new Heart().beat();
    }

}

# 1.2.4 匿名内部类

anonymous [əˈnɒnɪməs] 没有名字、隐藏名字 。将类的定义与类的创建放到一起完成

当实例仅使用一次的时候，此时，这个类的名字就不是很重要了。

适用场景：

  1. 只用到类的一个实例
  2. 类在定以后马上用到
  3. 给类命名并不会使代码更容易被理解

匿名内部类：

  1. 匿名内部类没有类型名称、实例对象名称
  2. 编译后的文件名：外部类$数字.class
  3. 无法使用private、public、protected、abstract、static修饰
  4. 无法便携构造方法，可以添加构造代码块
  5. 不能出现静态成员
  6. 匿名内部类可以实现接口，也可以继承父类，但不可兼得

根据传入的不同的人的类型，调用对应的read方法

先用非匿名方式实现

父类 抽象类

package com.lcs.tel;
public abstract class Person {
    private String name;
    public Person(){}

    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }

    public abstract void read();
}

子类继承父类抽象类，并实现父类抽象类里的方法

package com.lcs.tel;
// 一个类要继承一个抽象父类，需要实现父类的所有抽象方法
public class Man extends Person{
    @Override
    public void read() {
        System.out.println("man的read");
    }
}

package com.lcs.tel;

public class Women extends Person{
    @Override
    public void read() {
        System.out.println("women的read");
    }
}
