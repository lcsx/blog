# XML

# 1 XML作用与用途

全称 EXtensible Markup Language 【可拓展标记语言】

# 用途，如java程序的配置描述文件

# 2 XML语法规则

XML声明 XML声明说明XML文档的基本信息，包括版本号，字符集，写在第一行

<?xml version="1.0" encoding="UTF-8"?>

XML必须有根标签

<?xml ...?>不是根标签，ta下面第一个标签就是根标签，其他标签要放到根标签中

注释

 <!-- 员工信息 -->

处理特殊字符 标签内容中出现“<”，会破坏文档结构。

无效的xml：

<exam>
	<question>1+4<3是否正确?</question>
	<!-- 正确写法-->
	<question>1+4<&lt;3是否正确?</question>
</exam>
实体引用 对应符号 说明
< < 小于
> > 大于
& & 和号
' ' 单引号
" " 双引号

CDATA标签处理特殊字符 当有大量特殊字符出现时，通过实体引用的方式修改是对应符号是比较麻烦的。

可以通过CDATA标签处理大段包含特殊字符的文本.

<exam>
	<question>
		<![CDATA[ 1+4<3是否正确? ]]>
	</question>
</exam>

什么时候用属性表达 具有分类，标识性的

<?xml version="1.0" encoding="UTF-8"?>
<!-- 员工信息 -->
<hr>
    <employee no="1001">
        <name>duoduo</name>
        <age>18</age>
    </employee>
    <employee no="1002">
        <name>dandan</name>
        <age>12</age>
    </employee>
</hr>

验证xml是否正确 可以通过ide openin browser

# 3 XML语义约束

XML文档结构正确，但可能不是有效的。XML语义约束就是用于规定XML文档中允许出现哪些元素。

XML语义约束有两种定义方式：DTD 与 XML Schema

# 3.1 DTD

DTD（Document Type Definition 文档类型定义）

DTD文件的拓展名 “.dtd”

<?xml version="1.0" encoding="UTF-8" ?>
<!ELEMENT hr (employee+)>
<!ELEMENT employee (name,age,salary,department)>
<!ELEMENT name (#PCDATA)>
<!ELEMENT age (#PCDATA)>
<!ELEMENT salary (#PCDATA)>
<!ELEMENT department (dname,address)>
<!ELEMENT dname (#PCDATA)>
<!ELEMENT address (#PCDATA)>
<!ATTLIST employee no CDATA "">
dtd 解释
<!ELEMENT>标签 可以定义XML文档中允许出现的节点即数量
<!ELEMENT hr (employee)> 定义hr节点写只允许出现1个employee子节点
<!ELEMENT hr (employee)+> "+":最少出现1个employee子节点;"*":可出现0个或N个;"?":最多出现1个
<!ELEMENT employee (name,age,salary,department)> employee节点下必须包含以下四个节点，且按顺序出现
<!ELEMENT name (#PCDATA)> 定义name标签体之恶能是文本，#PCDATA代表文本元素
<!ATTLIST employee no CDATA ""> "no"属性名,CDATA:"no"是纯文本,""ta的默认值

# 3.2 XML引用DTD文件

在XML中使用<!DOCTYPE>标签引用DTD文件

<!DOCTYPE 根节点 SYSTEM "dtd文件路径">

<!DOCTYPE hr SYSTEM "hr.dtd">

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE hr SYSTEM "hr.dtd">
<!--公司员工数据-->
<hr>
    <employee no="3309">
        <name>张三</name>
        <age>31</age>
        <salary>4000</salary>
        <department>
            <dname>会计部</dname>
            <address>XX大厦-B103</address>
        </department>
    </employee>
    <employee no="3310">
        <name>李四</name>
        <age>23</age>
        <salary>3000</salary>
        <department>
            <dname>工程部</dname>
            <address>XX大厦-B104</address>
        </department>
    </employee>
</hr>

# 3.2 XML Schema

XML Schema比DTD更复杂，提供更多功能。提供数据类型、格式限定、数据范围等特性。

XML Schema文件拓展名“.xsd”

# 3.2.1 XML Schema规则

<?xml version="1.0" encoding="UTF-8" ?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
    targetNamespace="http://www.imooc.com/xml-schema" elementFormDefault="qualified">
    <element name="hr">
        <complexType>
            <sequence>
                <element name="employee" minOccurs="0" maxOccurs="99999">
                    <complexType>
                        <sequence>
                            <element name="name" type="string"></element>
                            <element name="age">
                                <simpleType>
                                    <restriction base="integer">
                                        <minInclusive value="18"></minInclusive>
                                        <maxInclusive value="65"></maxInclusive>
                                    </restriction>
                                </simpleType>
                            </element>
                            <element name="salary" type="integer"></element>
                            <element name="department">
                                <complexType>
                                    <sequence>
                                        <element name="dname" type="string"></element>
                                        <element name="address" type="string"></element>
                                    </sequence>
                                </complexType>
                            </element>
                        </sequence>
                        <attribute name="no" type="string" use="required"></attribute>
                    </complexType>
                </element>
            </sequence>
        </complexType>
    </element>
</schema>
标签 解释
complexType 声明复杂节点，不是单纯文本。即element name="hr"子节点是嵌套的结构就要
sequence 序列。要按顺序的
element name="employee" minOccurs="0" maxOccurs="99999" employee标签，最少出现0次，最多99999次
complexType 若子节点下还包含非单纯子节点，需要再用complexType
element name="dname" type="string" name里是string类型
simpleType 简单类型
restriction base="integer" 整数类型
minInclusive value=="18" maxInclusive 最小、大包含值18【含18】
minExclusive value=="18" maxInclusive 最小、大18【不包含18】
attribute 设置属性，name属性名，type值类型，use是否必填。注意attribute放的位置

# 3.2.2 XML Schema与XML关联

通过 命名空间 关联。

在.xsd文中

schema 标签中

targetNamespace属性定义一个命名空间 名称，约定俗成是一个网址，这个网址可以是不存在的。

elementFormDefault="qualified" 把当前.xsd文档当作默认文档生效。

在.xml文件中要设置三个项

  1. xmlns 当前文档命名空间
  2. xmlns:xsi 用于允许加载本地xsd文件
  3. xsi:schemaLocation 2.3几乎都是写si了
 <?xml version="1.0" encoding="UTF-8" ?>
<!--公司员工数据-->
<hr xmlns="http://www.imooc.com/xml-schema"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.imooc.com/xml-schema hr.xsd">
    <employee no="3309">
        <name>张三</name>
        <age>31</age>
        <salary>4000</salary>
        <department>
            <dname>会计部</dname>
            <address>XX大厦-B103</address>
        </department>
    </employee>
    <employee no="3310">
        <name>李四</name>
        <age>23</age>
        <salary>3000</salary>
        <department>
            <dname>工程部</dname>
            <address>XX大厦-B104</address>
        </department>
    </employee>
</hr>

在实际开发中，一般只会去引用外部的.xsd，不用自己写
