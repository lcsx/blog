# 1 数据统计分析

# 1.1 聚合函数

聚合函数用作简单的数据统计，如对数据求和、求最大最小值、平均值等。

聚合函数由五个，AVG平均数、SUM求和

# 1.1.1 AVG 平均数

求公司员工的平均月收入

SELECT AVG(sal+IFNULL(comm,0)) FROM t_emp AS 别名;

TIP

因为字段comm有null值，数字+null=null，所以要加个判断，IFNULL(),第二个参数即为 null值的替换值

# 1.1.2 SUM 求和

SUM只能用于数字型，字符型的求和结果=0，日期型求和=毫秒数相加

SELECT SUM(sal) FROM t_emp WHERE depton IN (10,20);

# 1.1.3 MAX 最大值

查询10和20部门中国，月收入最高的员工

SELECT MAX(sal+IFNULL(comm,0)) FROM t_emp WHERE deptno IN(10,20);

查询员工名字最长的是几个字符

SELECT MAX(LENGTH(ename)) FROM t_emp;

# 1.1.4 MIN 最小

SELECT MIN(hiredate) FROM t_emp;

# 1.1.5 COUNT 获取记录数

# COUNT(*) 获取所有的，包含空值的记录数
# COUNT(列名)获取非空值的记录数
SELECT COUNT(*),COUNT(ename) FROM t_emp;

查询1985年以后入职的员工，底薪超过公司平均底薪的员工数量

SELECT COUNT(*) FROM t_emp
	WHERE hiredate>="1985-01-01"
		AND sal>AVG(sal);

👆👆👆👆👆👆👆会报错

WARNING

聚合函数不能在WHERE子句中用。可通过HAVING来解决，本节下文会提到！

# 1.2 分组查询 GROUP BY

SELECT deptno,ROUND(AVG(sal)) AS "平均底薪" FROM t_emp GROUP BY deptno;

ROUND()四舍五入

# 1.2.1 逐级分组【分组内，再分组】

查询每个部门里，每种职位的任意数量和平均底薪

SELECT deptno,job,COUNT(*),AVG(sal) FROM t_emp
	GROUP BY deptno,job
		ORDER BY deptno;

TIP

查询语句中如果有GROUP BY语句，那么SELECT子句中的内容必须遵守：

SELECT子句只可有聚合函数，或者GROUP BY子句的分列组的那个字段

👇👇👇✔

SELECT deptno,COUNT(*),AVG(sal) FROM t_emp GROUP BY deptno;

👇👇👇❌

SELECT deptno,COUNT(*),AVG(sal),sal FROM t_emp GROUP BY deptno;

select子句中不能有

会报错因为group by中，sal既不是聚合函数，也不是用以gruop by子句的分组列。

# 1.2.2 对分组结果再次做汇总计算【类似excel 最后一行做汇总】

with rollup

SELECT deptno,COUNT(*),AVG(sal),MAX(sal),MIN(sal) FROM t_emp GROUP BY deptno WITH ROLLUP;

# 1.2.3 GROUP_CONCAT

GROUP_CONCAT 把某字段拼接成一个字符串，也是聚合了，就也在SELECT子句中用了

查询每个部门内底薪超过2000的人数 和 员工名字【把enamme字段拼接起来展示】

SELECT
	deptno,GROUP_CONCAT(ename),COUNT(*)
FROM t_emp
WHERE sal >=2000
GROUP BY deptno;

# 1.3 HAVING子句 【也是条件筛选】

HAVING子句

HAVING子句要配合GROUP BY使用，不能单独使用。

如果过滤条件中出现了聚合函数，则限制条件必须使用having 来替换 where，否则报错。

having 必须声明在group by 之后

# 1.3.1 HAVING子句用法

查询 平均底薪超过2000原的 部门编号

# 报错
SELECT deptno
FROM t_emp
WHERE AVG(sal)>=2000
GROUP BY deptno;

SELECT deptno
FROM t_emp
GROUP BY deptno HAVING AVG(sal)>=2000;

查询每个部门中，1982年后入职的员工超过2个人的部门编号

  1. 查询每个部门 --- 对部门分组
  2. 1982年后，普通条件，写在where中是可以的
  3. 超过2个人，COUNT(*)>2 。聚合函数，所以要用having.HAVING子句要配合GROUP BY使用，不能单独使用。
SELECT deptno FROM t_emp
WHERE hiredate>="1982-01-01"
GROUP BY deptno HAVING COUNT(*)>=2
ORDER BY deptno ASC;

GROUP BY HAVING中，不能拿某个字段即某个例去跟聚合函数判断，会报错。要解决这个问题，需要表连接

👇❌❌❌

SELECT deptno
FROM t_emp
GROUP BY deptno HAVING COUNT(*)>=2 AND sal>=AVG(sal);

另外，可以聚合函数比较一个具体的数据。

👇✔✔✔

SELECT deptno
FROM t_emp
GROUP BY deptno HAVING COUNT(*)>=2 AND AVG(sal)>=2000;

# 1.3.2 HAVING子句 特殊用法

GROUP BY 数字

SELECT deptno,COUNT(*) FROM t_emp GROUP BY 1;

即按deptno进行分组

having 和 where都是条件筛选，但having不能替代where

SELECT deptno,COUNT(*) FROM t_emp GROUP BY 1 HAVING deptno IN (10,20);

应该先用where筛选在进行分组，能提升查询效率

# 2 多表查询

从多张表中提取数据，必须指定关联的条件。如果不定义关联条件，就会出现无条件连接，两张表的数据会交叉连接，产生笛卡儿积。

笛卡尔积

# 没有指定连接条件
SELECT empno,name,dname FROME t_emp JOIN t_dept;
# 检索的结果数量有： t_emp数量*t_dept数量

建立表连接 JOIN ... ON 表1.字段= 表2.字段

SELECT e.empno,e.ename,d.dname 
FROM t_emp e JOIN t_dept d
ON e.deptno=d.deptno;

# 2.1 内连接

表连接分为两种：

  1. 内连接 【结果集种只保留符合连接条件的记录】【交集】
  2. 外连接 【不管符不符合连接条件，记录都要保留在结果集中】

内连接语法【三种】效率相当

第一种 用ON

# INNER 可以省略
SELECT ... FROM 表1
[INNER] JOIN 表2 ON 连接条件
[INNER] JOIN 表3 ON 连接条件
...;

第二种 WHERE

SELECT ... FROM 表1
[INNER] JOIN 表2 WHERE 连接条件
[INNER] JOIN 表3 WHERE 连接条件
...;

第三种

SELECT ... FROM 表1,表2 WHERE 连接条件;

# 2.1.1 查询每个员工的工号、姓名、部门名称、底薪、职位、工资等级

# 这里涉及三个表
# 工资等级表中只有工资范围，而员工工资只有一个具体的数据，具体数据对不上
# 但可以用 “范围”来关联表
SELECT e.empno,e.ename,d.dname,e.sal,e.job,s.grade
FROM t_emp e JOIN t_dept d ON e.deptno=d.deptno
JOIN t_salgrade s ON e.sal BETWEEN s.losal AND s.hisal;

内连接关联字段

内连接的数据表不一定必须有同名字段，只要字段之间符合逻辑关系就可以

# 2.1.2 查询底薪超过公司平均底薪的员工信息

👇👇👇❌❌❌

SELECT e2.empno,e2.ename,e2.sal
FROM t_emp e1 JOIN t_emp e2 ON e2.sal>=AVG(e1.sal);

ON和WHERE 可以互换的，所以ON子句也不能有聚合函数AVG，所以会报错；

SELECT e.empno,e.ename,e.sal
FROM t_emp e JOIN
(SELECT AVG(sal) avg FROM t_emp) t   #AVG(sal)字段起别名avg ；并将查询结果当作作表，别名t
ON e.sal>t.avg;

# 2.1.3 查询于SCOTT相同部门的员工有哪些

方法1：效率慢

SELECT ename
FROM t_emp
WHERE deptno=(SELECT deptno FROM t_emp WHERE ename="SCOTT")
AND ename!="SCOTT";
# 如果t_emp有1000条数据，会执行1000次(SELECT deptno FROM t_emp WHERE ename="SCOTT")查询

方法2：推荐表连接

SELECT  e2.ename
FROM t_emp e1 JOIN t_emp e2 ON e1.deptno=e2.deptno
WHERE e1.ename="SCOTT" AND e2.ename!="SCOTT";

同一数据表做表连接

# 2.1.4 查询RESEARCH部门的人数、最高底薪、最低底薪、平均底薪、平均工龄

SELECT COUNT(*),MAX(e.sal),MIN(e.sal),AVG(e.sal),

SELECT

# 2.1.6 查询每个底薪 超过部门底薪的员工信息

SELECT e.empno,e.ename,e.sal
FROM t_emp e JOIN
(SELECT deptno,AVG(sal) avg FROM t_emp GROUP BY deptno) t
ON e.deptno=t.deptno AND e.sal>t.avg;

# 2.2 外连接

内、外连接的区别

除了符合条件的记录之外，结果集中还会保留不合符条件的记录。 外连接还分为 1：左外连接；2：右外连接。

左连接和右连接

左外连接就是保留左表所有的记录，与右表作连接。如果右表右符合条件的记录就与左表连接。如果右表没有符合条件的记录，就用NULL去与左表连接。有链接同理。

# 2.2.1 查询每个部门的名称和部门的人数

SELECT 
	e.empno,e.ename,d.dname
FROM
	t_emp e
LEFT JOIN t_dept d ON e.deptno=d.deptno;

SELECT d.dname,COUNT(*),GROUP_CONCAT(e.ename)
FROM t_dept d LEFT JOIN t_emp e
ON d.deptno=e.deptno
GROUP BY d.deptno;

COUNT(*)

"COUNT(✳)"统计的是有效记录，有无null值都算一条有效记录。当左表中的记录在右表中无对应时，右表也用null与左表进行连接，所以，即使右表无记录与左表对应，COUNT(✳)的个数也是“1”.

但这个值不是想要的。解决：用右表的字段进行COUNT统计

SELECT d.dname,COUNT(e.deptno),GROUP_CONCAT(e.ename)
FROM t_dept d LEFT JOIN t_emp e
ON d.deptno=e.deptno
GROUP BY d.deptno;

# 2.2.2 查询每个部门的名称和部门的人数.如果没有部门的员工，部门名用null代替。【UNION】

UNION 合并多个表，要求每个表的字段、数量长度都一致可以。

(查询语句) UNION (查询语句) UNION (查询语句) ...

2.2.1中，有的记录的某些字段值并没有 筛选条件的字段，导致不会出现在统计的结果中.如“陈浩”的部门字段为空，ta就不会出现在以部门字段为分组的 结果集中。

(SELECT d.dname,COUNT(e.deptno),GROUP_CONCAT(e.ename)
FROM t_dept d LEFT JOIN t_emp e
ON d.deptno=e.deptno
GROUP BY d.deptno)
UNION
(SELECT d.dname,COUNT(*),GROUP_CONCAT(e.ename)
FROM t_dept d RIGHT JOIN t_emp e
ON d.deptno=e.deptno
GROUP BY d.deptno);

注意

表连接中，COUNT() 中的字段

外连接注意事项

内连接只保留符合条件的记录，所以查询条件写在ON子句和WHERE子句中的效果的相同的。

但是外连接里，条件写在WHERE子句里，不符合条件的记录是会被过滤掉的，而不是保留下来。

on 不符合条件的记录会保留

SELECT
e.ename,d.dname,d.deptno
FROM
t_emp e
LEFT JOIN t_dept d ON e.deptno=d.deptno
AND e.deptno=10;

where 不保留不符合条件的记录

SELECT
e.ename,d.dname,d.deptno
FROM
t_emp e
LEFT JOIN t_dept d ON e.deptno=d.deptno
WHERE e.deptno=10;

# 3 子查询

# 3.1 WHERE子查询

子查询是一种查询中嵌套查询的语句。普通查询分为 SELECT子句、FROM子句、WHERE子句，这些语句都是可以定义子查询的。

在前面的表连接学习中有提到 ：每条记录都会反复执行WHERE子句中的子查询，导致子查询效率低下。【反复的子查询成为相关子查询，WHERE子查询也是相关子查询】

但把FROM子查询只会执行一次，查询结果当作一张表来使用，跟其他表做连接，所以效率高。FROM子查询是推荐的。

WHERE子查询

SELECT emonp,ename,sal
FROM t_emp
WHERE sal>=(SELECT AVG(sal) FROM t_emp);

# 3.2 FROME子查询

SELECT e.empno,ename,e.sal,t.avg,e.deptno
FROM t_emp e JOIN 
(SELECT deptno,AVG(sal) avg 
FROM t_emp 
GROUP BY deptno) t
ON e.deptno=t.deptno AND e.sal<t.avg;

# 3.3 SELECT子查询

每输出一条记录的时候都要执行一次，查询效率低

SELECT 
e.ename,e.empno,(SELECT dname FROM t_dept WHERE deptno=e.deptno) "部门名称"
FROM t_emp e;

TIP

子查询能访问外面的数据表

# 3.4 单行/多行子查询

按照结果集数量划分，分为

  1. 单行子查询：结果集只有一条记录
  2. 多行子查询：结果集有多行记录【只能出现在WHERE和FROM子句中，不能出现在SELECT子句中】

# 3.4.1 查询FORD和MARTIN两个人的同事

方法一：WHERE方式

SELECT ename
FROM t_emp
WHERE deptno IN
(SELECT deptno FROM t_emp WHERE ename IN("FORD","MARTIN"))
AND ename NOT IN("FORD","MARTIN");

方式二：FROM

SELECT ename
FROM t_emp e
JOIN (SELECT deptno FROM t_emp WHERE ename IN("FORD","MARTIN")) t
ON e.deptno=t.deptno 
WHERE ename NOT IN("FORD","MARTIN");

# 3.4.2 IN、ALL、ANY、EXISTS关键字

WHERE子句中的多行子查询 查询比FORD和MARTIN底薪都高的员工信息

SELECT ename FROM t_emp
WHERE sal>ALL
(SELECT sal FROM t_emp WHERE ename IN("FORD","MARTIN"));

WHERE子句查询结果集有多行记录，WHERE子句每条记录通过ALL来与多行结果集进行判断。

SELECT ename,sal
FROM t_emp
WHERE EXISTS
(SELECT *
FROM t_salgrade
WHERE sal BETWEEN losal AND hisal
AND grade IN(3,4));

SELECT e.ename,s.grade
FROM t_emp e 
JOIN t_salgrade s
ON e.sal BETWEEN s.losal AND s.hisal;

SELECT e.ename,s.grade
FROM t_emp e 
JOIN t_salgrade s
ON e.sal BETWEEN s.losal AND s.hisal
# WHERE s.grade=3 OR s.grade=4;
WHERE s.grade IN(3,4);
